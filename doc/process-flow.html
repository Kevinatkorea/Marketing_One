  • 배치 ID 자동 부여 (벌크 등록 시)

✅ Phase A: 규칙 기반 사전검사 ▼
AI 호출 없이 키워드/구조 규칙으로 빠르게 사전 검증 (비용 절감)
Auto-Fail 가능 Guide 규칙 참조

검사 항목 (VerificationRule)

필수 키워드 (weight: 30) — 키워드 60% 이상 포함 여부
금지 키워드 (weight: 25, autoFail) — 금지어 발견 시 즉시 부적합
콘텐츠 구조 (weight: 15) — 이미지/링크/최소 글자수 검사
Auto-Fail 메커니즘
  • 규칙에 isAutoFail: true 설정된 항목이 실패 시
  • Phase B(AI 분석) 건너뛰고 즉시 부적합 판정
  • AI API 호출 비용 절감 효과

🏆 점수 산정 & 등급 부여 ▼
Phase A + B 결과를 합산하여 100점 만점 정규화 점수로 등급 판정
점수 계산 공식
  • totalScore = SUM(각 규칙 score)
  • maxScore = SUM(각 규칙 weight) (기본 100)
  • normalizedScore = (totalScore / maxScore) * 100
적합 (OK)
≥ 90
경고 (Warning)
75~89
부적합 (Fail)
< 75
저장 데이터
  • status: verified / failed
  • verification: { result, score, grade, details[], issues[], checkedAt }
  • verificationHistory: 매 검증마다 이력 추가

네이버 API로 실제 댓글을 수집하고 Claude AI로 감성/우선순위 분류

댓글 수집
  • 네이버 카페: v2.1 API 응답에 포함 (20개 이하) / v3 페이지네이션 (20+)
  • 기타: 크롤링 텍스트 기반 폴백
  • 작성자 닉네임 + 댓글 내용 구조화
AI 감성 분석 결과
  • 감성: positive / neutral / negative (-1.0 ~ 1.0)
  • 카테고리: 긍정반응, 일반질문, 광고의심, 불만, 거짓정보, 신상노출, 스팸
  • 우선순위: immediate → high → monitor → ignore
  • 대응 필요: responseRequired boolean

📊 대시보드 집계 & 표시 ▼
전체 바이럴 데이터를 집계하여 실시간 대시보드에 시각화
GET /dashboard/viral
집계 항목
  • 통계 카드: 총 바이럴, 검증완료, 적합률, 부정댓글, 마지막 검증일
  • 파이차트: 적합/경고/부적합 분포
  • 바차트: 부적합 원인별 카운트 (issues 집계)
  • 목록: 필터링 (전체/적합/경고/부적합/대기)
사용자 액션
  • 개별 검증 / 재검증 — 스피너 + "검증중" + 토스트 알림
  • 전체 재검증 — 진행률 표시 (2/5) + 완료 토스트
  • 댓글 수집 — 감성 분석 결과 뱃지 표시
  • 바이럴 삭제 — 확인 다이얼로그
