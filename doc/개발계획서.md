# 퍼포먼스 마케팅 관리시스템 - 개발계획서 (바이브코딩용)

> **문서버전**: v1.0 (랄프루프 5차 검증 완료)
> **작성일**: 2026-03-27
> **목적**: 바이브코딩 기반 개발을 위한 상세 개발계획서
> **검증**: 마케팅전문가 / 비즈니스 / UX / 기술 / 운영 5관점 검증 완료

---

## 1. 프로젝트 개요

### 1.1 시스템 명칭
**PerformanceOne** - 퍼포먼스 마케팅 통합 관리시스템

### 1.2 비전
퍼포먼스 마케팅팀의 바이럴, 광고, SEO, 콘텐츠 마케팅 등 전 채널을 하나의 대시보드에서 관리하고, 가이드 적합성 자동 검증 및 부정 댓글 모니터링까지 자동화하는 올인원 시스템

### 1.3 개발 우선순위 (Phase)

| Phase | 모듈 | 설명 | 우선순위 |
|-------|------|------|----------|
| **Phase 1** | 바이럴 관리 (카페 바이럴) | 프로젝트 개설, 가이드 등록, URL 등록, 적합성 검증, 댓글 모니터링 | 🔴 최우선 |
| **Phase 1.5** | 기본 리포팅 + 업체관리 | 성과 KPI, 업체/정산, 경영진 리포트 | 🔴 |
| Phase 2 | 광고 관리 | 네이버/구글/메타 광고 캠페인 통합 관리 | 🟡 |
| Phase 3 | SEO/GEO 관리 | 키워드 트래킹, 검색엔진 최적화 모니터링 | 🟡 |
| Phase 4 | 콘텐츠 마케팅 | 블로그/SNS 콘텐츠 관리 및 성과 추적 | 🟢 |
| Phase 5 | 통합 대시보드 | 전 채널 ROI 통합 리포트 | 🟢 |
| Phase 6 | 텔레그램 봇 연동 | 알림 자동화 (바이럴 검증결과, 부정댓글 등) | 🟢 |

## 2. 기술 스택

### 2.1 프론트엔드
- **React 18** + TypeScript
- **Tailwind CSS** (스타일링)
- **shadcn/ui** (UI 컴포넌트 라이브러리)
- **React Router v6** (라우팅)
- **Recharts** (차트/대시보드)
- **Zustand** (상태관리)
- **ExcelJS** (엑셀 파싱 - 일괄등록용)

### 2.2 백엔드
- **Node.js** + Express + TypeScript
- **JSON 파일 기반 DB** (초기 배포용 → 추후 MySQL 마이그레이션)
- **pino** (구조화된 로깅)
- **Zod** (입력 검증 스키마)
- **Bull/BullMQ** (작업 큐 - 배치 검증용, Phase 1.5)

### 2.3 DB 전략 (JSON → MySQL 마이그레이션 대비)
```
설계원칙: Repository Pattern 적용 + 트랜잭션 인터페이스 포함
├── /repositories/
│   ├── interfaces/        # DB 인터페이스 정의 (CRUD + batchInsert + 트랜잭션)
│   ├── json/             # JSON 파일 구현체 (Phase 1) + FileLocker 동시성 제어
│   └── mysql/            # MySQL 구현체 (추후)
```
- JSON 파일은 `/data/` 디렉토리에 테이블별로 분리
- Repository 패턴으로 DB 레이어 추상화 → MySQL 전환 시 구현체만 교체
- **FileLocker**: JSON 파일 동시 쓰기 방지 (락 메커니즘)
- **인덱싱**: 메모리 내 Map 기반 인덱스 (id, projectId, status 등)

### 2.4 외부 연동
- **Puppeteer + Stealth Plugin** (URL 콘텐츠 크롤링 - 봇탐지 회피)
- **Playwright** (백업 크롤러)
- **Claude API** (가이드 적합성 AI 검증 + 감성분석)
- **Telegram Bot API** (알림 전송 - Phase 6, 사전 인터페이스 준비)

## 3. Phase 1 상세 설계 - 바이럴 관리

### 3.1 기능 플로우

```
[메인] 프로젝트 목록
  ↓ 프로젝트 선택/생성
[프로젝트] 대시보드
  ↓ 바이럴 관리 진입
[바이럴 관리]
  ├── 📊 전체 현황 대시보드 (상단)
  │   ├── 총 바이럴 건수 / 검증완료 / 미검증
  │   ├── 적합(OK) vs 경고 vs 부적합(NO) 3단계 비율 차트
  │   ├── 부정 댓글 발견 건수
  │   ├── 상품/서비스별 바이럴 현황 (필터가능)
  │   └── 부적합 원인 분석 (사유별 비율)
  │
  ├── 📋 바이럴 목록 (하단 테이블)
  │   ├── 상품명 | 제목 | 카페명 | 검증상태(🟢🟡🔴) | 댓글상태 | 등록일
  │   └── 필터/정렬/검색 + 페이지네이션
  │
  ├── 📝 가이드 등록 (모달/페이지)
  │   ├── 상품/서비스 선택
  │   ├── 필수 포함 키워드 (가중치 설정)
  │   ├── 금지 키워드 (Auto-fail 설정 가능)
  │   ├── 톤앤매너 가이드
  │   ├── 필수 이미지/링크 포함 여부
  │   ├── 최소 글자수
  │   ├── 네이버 광고성 콘텐츠 정책 준수 여부
  │   ├── 기타 가이드라인 (자유텍스트)
  │   └── 가이드 복제/템플릿 저장 기능
  │
  ├── 🔗 바이럴 등록 (3가지 방식)
  │   ├── 방식1: 단건 등록 (모달)
  │   │   ├── 상품/서비스 선택 (가이드 자동 연결)
  │   │   ├── 바이럴 제목, URL, 카페명/플랫폼, 작성자/업체명
  │   │   └── 등록 즉시 검증 시작 옵션
  │   │
  │   ├── 방식2: ✨ 텍스트 붙여넣기 일괄등록 (NEW)
  │   │   ├── 텍스트 영역에 복사/붙여넣기
  │   │   ├── 자동 파싱 (카페명 / 제목 / URL 분리)
  │   │   ├── 지원 형식:
  │   │   │   ├── "카페명\n제목\nURL" (3줄 반복)
  │   │   │   ├── "카페명 | 제목 | URL" (한줄 형식)
  │   │   │   └── "카페명\t제목\tURL" (탭 구분)
  │   │   ├── 중복 URL 자동 감지 → 경고 표시
  │   │   ├── 파싱 미리보기 → 상품/가이드 일괄 선택 → 저장
  │   │   └── 등록 결과 리포트 (성공/중복/실패)
  │   │
  │   └── 방식3: ✨ 엑셀 파일 업로드 일괄등록 (NEW)
  │       ├── 엑셀 템플릿 다운로드 기능
  │       ├── 파일 드래그앤드롭 업로드
  │       ├── 자동 파싱 + 유효성 검사
  │       ├── 중복 URL 자동 감지 → 스킵 옵션
  │       └── 업로드 결과 리포트 (행번호별 성공/실패)
  │
  ├── ✅ 가이드 적합성 검증
  │   ├── URL 내용 자동 크롤링 (Stealth Puppeteer)
  │   ├── 2단계 검증: 사전 체크리스트 → AI 분석
  │   ├── 가중치 기반 점수 계산 (0-100)
  │   │   ├── 🟢 90-100점: 적합 (배포 가능)
  │   │   ├── 🟡 75-89점: 경고 (검토 필요)
  │   │   └── 🔴 0-74점: 부적합 (수정 필요)
  │   ├── Auto-fail 규칙: 금지키워드 사용 시 즉시 부적합
  │   ├── 부적합 사유 상세 리포트 + 재검증 요청 기능
  │   ├── 검증 이력 추적 (1차 NO → 수정 → 2차 OK)
  │   └── [추후] 텔레그램 봇 "바이럴" 토픽으로 전송
  │
  └── 💬 댓글 모니터링
      ├── URL별 댓글 자동 수집
      ├── 부정적 댓글 AI 감지 + 유형 분류
      │   ├── 즉시 대응 필요 (거짓정보, 신상노출)
      │   ├── 우선 대응 (불만, 답변필요 질문)
      │   ├── 모니터링 (추가 부정 패턴 감시)
      │   └── 무시 (광고성 의심)
      ├── 부정 댓글 하이라이트 표시 + 대응 이력 관리
      └── [추후] 텔레그램 알림 + AI 답변 초안 자동 생성
```

### 3.3 API 엔드포인트 설계

```
# 프로젝트
POST   /api/projects              - 프로젝트 생성
GET    /api/projects              - 프로젝트 목록
GET    /api/projects/:id          - 프로젝트 상세
PUT    /api/projects/:id          - 프로젝트 수정
DELETE /api/projects/:id          - 프로젝트 삭제

# 상품/서비스
POST   /api/projects/:pid/products         - 상품 등록
GET    /api/projects/:pid/products         - 상품 목록
PUT    /api/projects/:pid/products/:id     - 상품 수정
DELETE /api/projects/:pid/products/:id     - 상품 삭제

# 가이드
POST   /api/projects/:pid/guides           - 가이드 등록
GET    /api/projects/:pid/guides           - 가이드 목록
GET    /api/projects/:pid/guides/:id       - 가이드 상세
PUT    /api/projects/:pid/guides/:id       - 가이드 수정
POST   /api/projects/:pid/guides/:id/clone - 가이드 복제

# 바이럴
POST   /api/projects/:pid/virals           - 바이럴 단건 등록
GET    /api/projects/:pid/virals           - 바이럴 목록 (필터/페이지네이션)
GET    /api/projects/:pid/virals/:id       - 바이럴 상세
PUT    /api/projects/:pid/virals/:id       - 바이럴 수정
DELETE /api/projects/:pid/virals/:id       - 바이럴 삭제
POST   /api/projects/:pid/virals/:id/verify    - 적합성 검증 실행
POST   /api/projects/:pid/virals/:id/comments  - 댓글 수집 실행

# ✨ 일괄등록 (NEW)
POST   /api/projects/:pid/virals/bulk-text     - 텍스트 붙여넣기 일괄등록
POST   /api/projects/:pid/virals/bulk-excel    - 엑셀 파일 일괄등록

GET    /api/projects/:pid/virals/export-template  - 엑셀 템플릿 다운로드

# 대시보드
GET    /api/projects/:pid/dashboard/viral  - 바이럴 대시보드 통계
```

### 3.4 페이지 구조 (라우팅)

```
/                              → 프로젝트 목록 (메인)
/projects/new                  → 프로젝트 생성
/projects/:id                  → 프로젝트 대시보드
/projects/:id/viral            → 바이럴 관리 (대시보드 + 목록)
/projects/:id/viral/guides     → 가이드 관리
/projects/:id/viral/guides/new → 가이드 등록
/projects/:id/viral/new        → 바이럴 URL 등록 (단건)
/projects/:id/viral/bulk       → ✨ 바이럴 일괄등록 (텍스트/엑셀)
/projects/:id/viral/:vid       → 바이럴 상세 (검증결과 + 댓글)
```

```
   → Auto-fail 위반 시: AI 호출 스킵, 즉시 🔴 판정

4. [Phase B: AI 분석] (사전 체크 통과 시에만 실행)

7. [추후] 텔레그램 봇 토픽 "바이럴"로 결과 전송
```

## 부록: 바이브코딩 가이드

### 프롬프트 전략
각 개발 단계에서 AI에게 전달할 핵심 컨텍스트:

1. **프로젝트 초기 설정**: 이 개발계획서 전체를 컨텍스트로 제공
2. **컴포넌트 개발**: 해당 섹션의 데이터 모델 + API + UI 레이아웃 제공
3. **일괄등록 개발**: 텍스트 파싱 알고리즘 + 입력 형식 예시 제공
4. **검증 엔진 개발**: 2단계 검증 프로세스 + 가중치 규칙 스키마 제공
5. **테스트**: API 엔드포인트 목록 + 예상 입출력 제공

### 코딩 컨벤션
- 컴포넌트: PascalCase (`ViralDashboard.tsx`)
- 함수/변수: camelCase (`fetchViralList`)
- API 라우트: kebab-case (`/api/viral-management`)
- 상수: UPPER_SNAKE_CASE (`MAX_RETRY_COUNT`)
- 한글 주석 적극 활용 (비즈니스 로직 설명)
- 파일당 단일 책임 원칙 (바이브코딩 시 컨텍스트 명확화)
